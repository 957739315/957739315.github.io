vue源码分析

笔记总结

准备工作

代码规范

  • 使用 Flow(类似 TypeScript ) 做静态类型检查，具体相关配置在 .flowconfig 文件中
  • vuejs 是一个跨平台的 MVVM 框架

源码构建

  • 主要源码目录结构 src：compiler(编译相关)、core(核心代码)、platforms(不同平台的支持，web 平台和配合 weex 跑在 natvie 客户端)、server(服务端渲染)、sfc(.vue文件解析)、shared(共享代码)
  • 使用 Rollup(类似 webpack) 来进行源码构建，保存到 dist 目录下
  • 支持不同格式的构建，比如 CommonJS 规范、UMD 规范、ES Module 规范，默认使用 UMD
  • 源码构建的入口文件是 scripts/build.js
  • 使用 vue-cli 初始化 vue 项目，提供 Runtime Only(运行时不编译) 和 Runtime+Compiler(运行时编译) 两个版本，此处的编译指将 template 编译成 render 函数

如何完成 vue 的初始化过程？？

  • 入口文件 src/platforms/web/entry-runtime-with-compiler.js，通过 import Vue from ...找到 vue的构造函数（实例化方法），找到最终 vue 构造函数的路径为 src/core/instance/index.js;
  • 通过 xxxMixin 的调用往 Vue 的 prototype 上挂载一些方法
  • 通过 initGobalAPI 挂载了很多全局的静态方法

数据驱动

所谓数据驱动，是指视图是由数据驱动生成的，我们对视图的修改，不会直接操作 DOM，而是通过修改数据。

new Vue 发生什么？

Vue 初始化主要就干了几件事情，合并配置，初始化生命周期，初始化事件中心，初始化渲染，初始化 data、props、computed、watcher 等等。

vue 挂载 $mount 怎么实现？

  • 对 el 做了限制，Vue 不能挂载在 body、html 这样的根节点上
  • 如果没有定义 render 方法，则会把 el 或者 template 字符串转换成 render 方法
  • 只有调用 compileToFunctions 方法，渲染成 render 函数才会调用 mountComponent 挂载组件
  • mountComponent 核心就是先实例化一个渲染 Watcher，在它的回调函数中会调用 updateComponent 方法，在此方法中调用 vm._render 方法先生成虚拟 Node，最终调用 vm._update 更新 DOM。函数最后判断为根节点的时候设置 vm._isMounted 为 true， 表示这个实例已经挂载了，同时执行 mounted 钩子函数。 这里注意 vm.$vnode 表示 Vue 实例的父虚拟 Node，所以它为 Null 则表示当前是根 Vue 的实例。

render 函数

vm._render 是通过执行 createElement 方法并返回虚拟 node（vnode）。
执行 initRender 方法之前，做了一个拦截 initProxy，判断是否存在变量未在 data 中定义的，则报错。注意此拦截，只在开发环境做，生产环境下不拦截判断。

Virtual DOM

  • Virtual DOM 除了它的数据结构的定义，映射到真实的 DOM 实际上要经历 VNode 的 create、diff、patch 等过程。
  • Virtual DOM 是借鉴了一个开源库 Snabbdom

createElement 实现原理

  1. children 的规范化：把 children 变成一个类型为 VNode 的 Array

    • 把 vnode 节点的子节点页都规范成 vnode 类型
    • Virtual DOM 实际是一个树状结构，每个 VNode 可能会有若干个子节点，每个子节点页应该是 VNode 类型。
    • normalizeChildren(children) 和 simpleNormalizeChildren(children) 方法都是为了将子节点转化为一维的 vnode。
    • 调用 simpleNormalizeChildren(children) 场景：针对函数式组件（包含有组件的子节点），将嵌套的子节点变为一维数组
    • 调用 normalizeChildren(children) 场景：针对还有嵌套数组的组件（eg. v-for、<template>、<slot>），把多层嵌套的子节点变成一维数组

  2. 创建一个 VNode 实例

    • 内置节点，则创建普通 VNode
    • 已注册的组件名，则使用 createComponent 创建一个组件类型的 VNode

update

  • 调用时机：首次渲染、数据更新时
  • vm._update 核心调用 vm.__patch__
  • 在真实浏览器下，会通过 patch 方法转换成 DOM；在服务端渲染下，不会把 VNode 转换成 DOM，只是返回一个空函数
  • createChildren 方法原理：先遍历虚拟子节点，递归调用 createElm 方法（采用深度优先遍历算法）去创建真实的 DOM （直到深入 text 文本节点）并调用 inset 方法插入到父节点中（插入顺序是先子后父）。
  • 不断插入到父节点，直到插入到 body 节点，然后渲染新的 DOM 节点，删除旧的 DOM 节点。
  • 创建一个新节点，然后把旧的节点替换了。

组件化

createComponent 组件创建生成

  1. 构造子类构造函数

组件构造器的生成，是继承了 Vue。
通过 Vue.extend 函数返回子构造器，其中在 vue.extend 函数中，会校验组件名、将子构造器原型指向父构造器原型、合并 options、初始化 props、初始化 computed、初始化静态方法等等。

实际上，通过原型继承的方式返回每个组件的构造器。

优点：通过使用 cachedCtors 对象缓存所有初始化过的组件，方便下次调用时，直接返回初始化成功后的组件构造器，提高效率。

  1. 安装组件钩子函数

通过 installComponentHooks 函数安装组件的 hooks

在初始化一个 Component 类型的 VNode 过程中，合并了 componentVNodeHooks 钩子函数到 VNode.data 中，便于在最终执行时依次执行这两个钩子函数。其中，componentVNodeHooks 实现了 init、prepatch、insert、destroy 函数。

  1. 实例化 VNode

不同于普通元素节点的 vnode（在 createElement 中有对参数 tag 的判断是否为字符串，若是则实例化一个普通 VNode 节点，否则通过 createComponent 创建一个组件 VNode)，组件 vnode 没有 children

接着，通过 vm._update -> patch 方法将 VNode 转化为真正 DOM 节点，如何做的呢？？

patch

  1. 组件 patch 整体流程

patch 调用 createElem 创建节点，再调用 createComponent 方法

遇到子组件就先 patch，直到不再遇到子组件，就开始 inset，先 inset 子组件，再 inset 父组件。

1）根组件 APP，转化为 VNode，其 parentEle 是一个 body

2）进入 createEle 方法进行初始化组件实例，再进入 createComponent 方法，再进入 createComponentInstanceForVnode 方法，创建组件实例 new Vue，挂载一些 option 和建立组件的父子关系 initLifecycle
3）进入子组件实例挂载 $mount

MVVM框架

MVVM是Model-View-ViewModel的简写。

Model：应用程序中用于处理应用程序数据逻辑的部分，它主要负责网络请求，数据库处理，I/O等的操作。

View：应用程序中处理数据显示的部分。在Android开发中，它一般对应着xml布局文件、Activity、Fragment。

ViewModel ：创建关联，将Model和View绑定起来。这样，一旦Model发生更改，ViewModel就会立即反馈给View从而自动刷新界面。
