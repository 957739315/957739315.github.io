js实现队列-先入先出

发表于 2019-12-02 分类于 前端开发 阅读次数：

什么是队列？

队列主要特点是先入先出（FIFO，First In First Out），它是一种操作受限的线性表数据结构，其中，入队是将一个数据放到队列尾部（插入元素）；出队是从队列的头部取出一个元素（删除元素）。

队列应用有循环队列、阻塞队列、并发队列、优先级队列等。

顺序队列

需要头指针和尾指针，尾指针指向一个新的空闲位置

缺点

问题1： 随着起始指针的移动，浪费了越来越多的空间。

分析：
假如可分配长度为 5 的数组，当我们当前入队了 4 个元素，再入队一个元素可以成功，然后无法再入队元素。
若出队前 4 个元素，此时理论上能再入队 4 个元素。但是在顺序队列中，头指针指向了最后一个元素空间，造成无法入队元素。

解决方法：将队列当前剩下的数据往前迁移。

出队时不进行数据搬移，虽然会导致数组的不连续。入队时，当没有空闲位置时，也就是 tail == n 入队时，再进行数据搬移，这样也就保持了数组的连续性，同时也解决了频繁的入队、出队操作，导致的频繁的数据搬移。

问题2： 采用数据往前迁移，存在问题是时间复杂度从 O(1) 变成了 O(n)

分析：当队列的 tail 队尾标志位，移动到数组的最右边后，如果有新的数据入队，将 head - tail 之间的数据，整体搬移到数组中的 0 - （tail-head）的位置

解决方法：使用循环队列

循环队列

循环队列可以看成一个圆环，也需要头指针和尾指针，尾指针指向一个新的空闲位置，如下图，

优点是，循环队列避免了数组的搬移操作。

难点是，在于如何确定队空和队满的判定条件以及 head 和 tail 的变化。

思考一下，tail 的变化，当 tail=7，如何让 tail = 0 呢？
可以发现 tail = (tail+1)%size， 那么 head 同理也遵循这样的变化 head=(head+1)%size。

再思考一下，如何判定队满呢？
可以发现规律， (tail+1)%size == head 的时候，队列就满了，但是 tail 指向的位置实际上是没有数据的，这就意味这循环队列浪费了一个存储空间，以空间换取时间。

链表队列

基于链表实现队列，也需要队头和队尾两个指针，头指针和尾指针，尾指针指向最后一个元素，相比前两种队列，其优点是支持无限排队的无界队列。

阻塞队列

什么是阻塞队列
队列可以看成一个商场，入队就是生产者生产商品放入商场中，出队就是消费者从商场中购买走商品。
当入队的速度大于出队的速度，会出现整个商场满货状态，那么这时，需要暂停入队，将生产者线程阻塞，直到商场出现货物空缺。
当出队的速度大于入队的速度，会出现整个商场缺货状态，那么这时，需要暂停出队，将消费者线程阻塞，直到商场有货物。

在多线程环境中，会出现若干个生产者，若干个消费者，所以队列实现了很好的数据共享，就像商场为解决生产和消费之间的关系提供了方式。

阻塞队列的分类

  1. 基于数组的阻塞队列 ArrayBlockingQueue
    需要维护一个定长数组、头指针、尾指针，其中，入队、出队共用同一个锁对象，因此，实际上无法真正并行运行。

  2. 基于链表的阻塞队列 LinkedBlockingQueue
    需要维护一个缓冲队列，当队列缓冲区达到最大值时，阻塞入队，直到出队一份数据，才唤醒入队进程。LinkedBlockingQueue 入队、出队分别采用
    独立锁来控制数据同步，这使得在高并发情况下，能够入队、出队并发运行，提高整个队列的并发性能。

  3. 延迟队列 DelayQueue
    只有在指定的延迟时间之后，才能取到元素。DelayQueue 没有限制队列大小，入队永远不会被阻塞，只有出队会被阻塞。

  4. 基于优先级的阻塞队列 PriorityBlockingQueue
    PriorityBlockingQueue 不会阻塞入队，而且只有队列为空时，才会阻塞出队。同时，使用的前提要求是，入队的速度决不能快于出队的速度。

  5. 同步队列 SynchronousQueue
    是一种无缓冲的等待队列，类似缺少了商场这个中间商，直接生产者和消费者面对面交易。

存在两种模式 公平模式 和 非公平模式：

  • 公平模式：采用公平锁，使用一个FIFO队列来阻塞多余的入队和出队操作，从而体系整体的公平策略；
  • 非公平模式：采用非公平锁，也使用一个FIFO队列来阻塞多余的入队和出队操作，但是当入队和出队速度存在差异，则会存在饥渴现象，即某些生产者或消费者的数据永远得不到处理。
